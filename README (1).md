# AI-Powered Task & Knowledge Management System

A full-stack MVP that lets an **Admin** build a searchable knowledge base and assign tasks, while **Users** perform AI-powered semantic search over documents and complete their assigned work — all secured with JWT authentication and role-based access control (RBAC).

Built as a Python Full Stack assignment focused on system design: clean architecture, a normalized relational schema, and a semantic search pipeline implemented from first principles (embeddings + FAISS) rather than a thin wrapper around an LLM API.

## Features

- 🔐 **JWT Authentication & RBAC** — Admin and User roles, with every API route protected accordingly.
- 📄 **Document Upload & Knowledge Base** — Upload `.txt` documents with metadata tracked in MySQL.
- 🔎 **AI-Powered Semantic Search** — Text is converted into vector embeddings (Sentence Transformers) and indexed in **FAISS** for fast similarity search — the retrieval logic is implemented directly, not outsourced to an external LLM API.
- ✅ **Task Management** — Admins create and assign tasks; Users track and update status (`Pending → Completed`).
- 🧮 **Dynamic Filtering** — Query tasks by status, assignee, and other fields (`/tasks?status=completed`).
- 🧾 **Activity Logging** — Every login, search, upload, and task update is recorded for auditability.
- 📊 **Analytics Dashboard** — Total tasks, completed vs. pending breakdown, and most-searched queries.

## System Architecture

```
                    User
                      │
                      ▼
              React Frontend
                      │
          REST API (JWT Authentication)
                      │
                      ▼
               FastAPI Backend
        ┌─────────────┼─────────────┐
        │             │             │
        ▼             ▼             ▼
 Authentication   Task Module   Document Module
        │             │             │
        └─────────────┼─────────────┘
                      ▼
              Embedding Service
                      │
                      ▼
                 FAISS Vector DB
                      │
                      ▼
                Semantic Search
                      │
                      ▼
                  MySQL Database
```

**Flow:** the React client authenticates via JWT and talks to FastAPI over REST. FastAPI routes requests to the Auth, Task, and Document modules. Uploaded documents are converted into embeddings and indexed in FAISS; search queries follow the same embedding path and are matched against that index. All structured data — users, tasks, documents, and activity logs — is persisted in MySQL.

## Database Schema

Core tables (see `database.sql` for full DDL):

| Table | Purpose |
|---|---|
| `users` | User accounts, hashed passwords, role reference |
| `roles` | Role definitions (`admin`, `user`) |
| `tasks` | Task records with status, assignee (FK → `users`), timestamps |
| `documents` | Uploaded document metadata, uploader (FK → `users`) |
| `activity_logs` | Audit trail — login, search, upload, task update events |

All relationships are enforced with primary/foreign keys and normalized to avoid redundant data.

## Installation

### Backend Setup

```bash
# 1. Clone the repository
git clone <repository-url>
cd <project-folder>/backend

# 2. Create and activate a virtual environment
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate

# 3. Install dependencies
pip install -r requirements.txt

# 4. Configure your MySQL connection
#    Update the database URL/credentials in the backend config (e.g. .env or config.py)

# 5. Import the schema
mysql -u <username> -p < database.sql

# 6. Start the FastAPI server
uvicorn main:app --reload
```

Visit **`http://localhost:8000/docs`** to confirm the backend is running via Swagger UI.

### Frontend Setup

```bash
# In a new terminal
cd <project-folder>/frontend

npm install
npm start
```

The app will open at **`http://localhost:3000`**.

## Running the Project

1. Start MySQL Server.
2. Start the FastAPI backend (`uvicorn main:app --reload`).
3. Start the React frontend (`npm start`).
4. Log in with valid credentials.
5. Access the dashboard → manage tasks → upload documents → run AI semantic search.

## Design Notes

- **Layered backend** — routes stay thin; business logic lives in `services/`, keeping controllers, models, and logic cleanly separated.
- **Semantic search implemented, not outsourced** — embeddings are generated locally with Sentence Transformers and indexed in FAISS, satisfying the "core logic must be implemented" requirement rather than relying solely on an external LLM API.
- **RBAC enforced at the route level** — every protected endpoint checks the caller's role via the JWT payload before executing.
- **Auditability by default** — the `activity_logs` table captures logins, searches, uploads, and task updates for traceability.
